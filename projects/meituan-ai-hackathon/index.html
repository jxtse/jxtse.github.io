Under the hood

技术架构

FastAPI + 自研 ReAct AgentRuntime + 统一 dataclass 工具层 + Mock 美团/点评数据。 Harness 全部收敛在 prompts.py，多阶段（规划 / 执行）零成本切换。

🧠 Agent 与 Harness

  • 自研 AgentRuntime ReAct 循环，system_prompt 支持 callable，规划/执行阶段动态切换
  • 工具统一 dataclass：name / description / parameters / invoke()，event_bus 推流前端
  • LLM 接口 OpenAI 兼容，线上版接 kimi-k2.6（api.moonshot.ai/v1）

🛡️ 三类异常自愈

  • no_table 无座 → 同人群/饮食约束切换可订餐厅
  • sold_out 无票 → 同类顺路活动替换，保留时间轴
  • conflict 冲突/超窗 → 触发规划器整体重排

⏱️ 硬性指标（赛题约束）

  • 方案生成 < 30s ✓ · 单工具 < 3s ✓ · 端到端 < 2min ✓（实测平均 548ms）
  • 至少处理 3 类异常 ✓ · Pass@1 ✓
  • 商户数据全 Mock，离线可演示，不接真实美团

🔌 为什么能纯静态展示

  • 本页是真实后端的离线回放：预设场景跑通 Kimi K2.6 后把回执烤进页面
  • 不在前端暴露任何 API Key，零泄露风险
  • 完整可交互版本需运行 FastAPI 后端 + Kimi 网关
